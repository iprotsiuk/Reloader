## Status Legend [v0.1]

- `Implemented` = present in repository with concrete runtime/tests evidence.
- `In Progress` = partial implementation exists, but acceptance criteria are not complete.
- `Planned` = target exists in scope docs, but implementation is not complete.

## Milestone Plan [v0.1/demo]

### Milestone M0 - Runtime Foundation (Completed)

Completed outcomes:
- World scene topology and bootstrap entry flow established.
- MainTown <-> IndoorRange round-trip travel implemented and regression-tested.
- Core persistence slice for inventory/weapons/world-object state implemented.
- Basic inventory and trade flow wiring operational in runtime architecture.

Primary evidence:
- `RoundTripTravelPlayModeTests`
- `WorldObjectStateSaveModuleTests`
- `EconomyControllerCheckoutPlayModeTests`

### Milestone M1 - Demo Core Loop (Current Target)

Target outcome:
- Player can accept one procedural assassination contract, prepare a rifle/load/optic, optionally validate the setup, kill a distant target, and escape police attention long enough to cash out.

Exit criteria status (2026-03-06):
- Single-stage press interaction criterion is met with explicit end-to-end PlayMode acceptance coverage.
- Range/group-size validation criterion is met and reflected as `Implemented` in this board.
- Contract-generation, police-heat, and arrest/death consequence criteria are still `Planned`.
- Save/load demo-loop criterion remains in progress pending remaining non-workbench module targets and future contract/heat state coverage.
- MainTown authored scene-default smoke checks remain open and now need contract-loop authoring coverage rather than range-only smoke coverage.

Integration snapshot:
- Workbench mount graph slice is integrated on `main`.
- Scoped PiP optics runtime is integrated on `main` after PR `#23`.
- Existing world, persistence, and trade slices provide the current foundation for the contract vertical slice.

### Milestone M2 - Demo Hardening (Next)

Target outcome:
- Stable candidate for external demo sessions.

Exit criteria:
- Regression suites for travel, inventory, persistence, weapons, trade, and contract loop run clean on the release branch.
- Docs and plans are synchronized: no checklist says "planned" for already-shipped work.
- Demo script (accept contract -> prep loadout -> shoot -> escape -> cash out -> save/load -> continue) is reproducible without editor intervention.
